# Phase 4 – Business Pipeline & Analytics using PySpark

## Overview

This project demonstrates the development of an end-to-end business analytics pipeline using **PySpark**. It covers the complete data engineering workflow, including data ingestion, cleaning, transformation, integration, aggregation, customer segmentation, and reporting. Each business problem is solved using both **Spark SQL** and the **PySpark DataFrame API**, showcasing how traditional SQL operations translate into distributed data processing with Apache Spark.

---

# Objectives

- Build an end-to-end ETL and business analytics pipeline
- Perform data cleaning and validation before analysis
- Integrate multiple datasets using joins
- Generate meaningful business insights from transactional data
- Implement equivalent solutions using Spark SQL and PySpark DataFrames
- Create a reusable reporting workflow

# Project Workflow

## 1. Data Ingestion

Load datasets into Spark DataFrames.

- Read customer dataset
- Read sales dataset
- Inspect schema
- Verify record counts

---

## 2. Data Cleaning

Perform preprocessing before analytics.

Tasks include:

- Remove records with null customer IDs
- Remove duplicate rows
- Filter invalid transaction amounts
- Handle missing values
- Validate data schema

---

## 3. Data Integration

Combine customer and sales datasets using the **customer_id** key to create a unified analytical dataset.

Segment customers based on total spending.

# SQL and PySpark Implementation

Each business problem is implemented using two approaches:

### Spark SQL

- Filtering
- Joins
- Aggregations
- CASE WHEN
- GROUP BY
- ORDER BY

### PySpark DataFrame API

- filter()
- select()
- join()
- groupBy()
- agg()
- withColumn()
- orderBy()
- when()
- Window Functions

This demonstrates the equivalence between SQL queries and PySpark transformations.
